# 8086 Assembly Student Record Manager

A robust, low-level management system designed for the **Computer Organization and Assembly Language (COAL)** curriculum. This project demonstrates direct hardware interaction and memory management using 16-bit x86 architecture.

## 🚀 Features
- **Persistent Storage:** Saves student records (Roll, Name, Sem, CGPA) to a `student.txt` file using DOS interrupts.
- **Data Persistence:** Implements file appending logic so records are not lost when the program closes.
- **Dynamic Memory Access:** Uses offset calculations to manage data arrays for up to 10 students.
- **User-Friendly Interface:** Menu-driven CLI for adding and displaying records.

## 🛠️ Technical Implementation
- **Architecture:** x86 (16-bit)
- **Language:** Assembly (ASM)
- **Interrupts Used:**
  - `INT 21h / AH=3Ch`: Create file
  - `INT 21h / AH=3Dh`: Open existing file
  - `INT 21h / AH=40h`: Write to file
  - `INT 21h / AH=42h`: File pointer manipulation (Append mode)
- **String Handling:** Custom `readString` and `writeStringToFile` procedures to handle variable-length inputs.

## 📂 Project Structure
- `PROJECT.ASM`: Core source code containing data segments and procedures.
- `student.txt`: Output file generated by the system to store student data.
